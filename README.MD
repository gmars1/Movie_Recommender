## Предварительные требования

Перед началом работы убедитесь, что у вас установлены следующие компоненты:

- **Python** (рекомендуется версия 3.8+)
- **pip** (менеджер пакетов Python)
- **Git**
- **Docker**
- **Docker Compose** (рекомендуется для управления Docker-контейнерами)

## Установка и запуск

### Клонирование репозитория

```bash
git clone https://github.com/Akhmat0/Movie-Recommender.git
cd Movie-Recommender-main
```

### Настройка виртуального окружения

Рекомендуется использовать виртуальное окружение для изоляции зависимостей проекта:

```bash
python -m venv venv
```

Активируйте виртуальное окружение:

-   **Windows:**
    ```bash
    venv\Scripts\activate
    ```
-   **macOS/Linux:**
    ```bash
    source venv/bin/activate
    ```

### Установка зависимостей

Установите необходимые Python пакеты из файла `requirements.txt`:

```bash
pip install -r requirements.txt
```

> **Примечание:** Убедитесь, что файл `requirements.txt` существует в корневой директории проекта и содержит все необходимые зависимости. Если его нет, вам может потребоваться создать его на основе используемых в проекте библиотек.

### Настройка базы данных (PostgreSQL с Docker)

Проект использует базу данных PostgreSQL. Самый простой способ настроить ее локально — использовать Docker.

1.  **Создайте файл `docker-compose.yml`** в корневой директории проекта со следующим содержимым:

    ```yaml
    version: '3.8'

    services:
      db:
        image: postgres:13 # Вы можете выбрать другую версию PostgreSQL при необходимости
        container_name: movie_db_container
        restart: always
        environment:
          POSTGRES_USER: postgres      # Должен совпадать с DB_FILMS_USER из .env
          POSTGRES_PASSWORD: postgres  # Должен совпадать с DB_FILMS_PASSWORD из .env
          POSTGRES_DB: movie_db        # Должен совпадать с DB_FILMS_NAME из .env
        ports:
          - "5430:5432"               # Маппинг порта: <хост_порт>:<контейнер_порт>
        volumes:
          - postgres_data:/var/lib/postgresql/data

    volumes:
      postgres_data:
    ```

2.  **Запустите Docker-контейнер** с базой данных:

    ```bash
    docker-compose up -d
    ```
    Это запустит контейнер PostgreSQL в фоновом режиме. База данных будет доступна на `localhost:5430`.

### Настройка переменных окружения

Приложение использует переменные окружения для конфигурации. Создайте файл `.env` в корневой директории проекта и добавьте в него следующие переменные:

```env
DB_FILMS_USER=postgres
DB_FILMS_PASSWORD=postgres
DB_FILMS_NAME=movie_db
DB_FILMS_HOST=localhost
DB_FILMS_PORT=5430

# DB_FILMS_CONTAINER_PORT=5432 # Информационная переменная, указывает на внутренний порт контейнера БД

# Добавьте другие необходимые переменные окружения для вашего приложения, например:
# SECRET_KEY=your_very_secret_key_here # Важно для безопасности веб-приложений
# DEBUG=True # Для режима разработки
```

> **Важно:**
> - Значения `DB_FILMS_USER`, `DB_FILMS_PASSWORD`, `DB_FILMS_NAME` в файле `.env` должны совпадать с переменными `POSTGRES_USER`, `POSTGRES_PASSWORD`, `POSTGRES_DB` в `docker-compose.yml`.
> - `DB_FILMS_HOST` должен быть `localhost`, так как приложение (запущенное на хосте) будет подключаться к базе данных, порт которой проброшен на `localhost`.
> - `DB_FILMS_PORT` должен быть `5430` (порт на хосте, указанный в `ports` в `docker-compose.yml`).
> - Файл `.env` не должен добавляться в систему контроля версий (Git), если он содержит чувствительные данные. Добавьте `.env` в ваш файл `.gitignore`.

### Миграции базы данных

Если ваш проект использует ORM (например, Django ORM, SQLAlchemy с Alembic), вам нужно будет применить миграции для создания таблиц в базе данных.

```bash
# Пример для Django:
# python manage.py makemigrations
# python manage.py migrate

# Пример для Flask-Migrate (Alembic):
# flask db init  # (если инициализируете в первый раз)
# flask db migrate -m "Initial database setup"
# flask db upgrade
```
> **Примечание:** Замените эти команды на те, которые актуальны для вашего проекта.

### Запуск приложения

Запустите веб-сервер разработки. Команда зависит от используемого фреймворка:

```bash
# Пример для Django:
# python manage.py runserver

# Пример для Flask:
# flask run --host=0.0.0.0 --port=8000

# Пример для FastAPI с Uvicorn:
# uvicorn main:app --reload --host 0.0.0.0 --port 8000
```
> **Примечание:** Замените эту команду на актуальную для вашего проекта. Укажите правильный файл и экземпляр приложения (например, `main:app` для FastAPI).

## Использование

После успешного запуска приложения откройте веб-браузер и перейдите по адресу:
`http://localhost:PORT` (где `PORT` — это порт, на котором запущено ваше приложение, например, `8000`).

## Остановка Docker-контейнера

Чтобы остановить и удалить контейнер с базой данных (если он был запущен через `docker-compose`):

```bash
docker-compose down
```
Если вы хотите просто остановить контейнер без удаления тома с данными:
```bash
docker-compose stop db
```
